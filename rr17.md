
Unpredictable Black Boxes are Terrible Interfaces

Agrawala's main point is that text-based generative AI interfaces are hard for users to steer. We may have a clear picture of what we want the output to be (or at least are able to tell the difference between a good and bad output), and we may think that our prompt provides clear instructions, but the output does not always align with our expectation. The reason for this is because the interface is stochastic and black-boxed---the same input will not always produce the same output, and user conceptual models of the input-output mapping does not align with the true system model.

It's well-documented that tiny changes to user prompts can lead to vastly different outputs, even if the prompts bare essentially the same meaning (https://aclanthology.org/2024.findings-naacl.130/). The more complex part that this blog post discusses is the difficulty of knowing how to tune your prompt when the zero-shot is wrong. 

I think the average LLM user's conceptual model is, more or less, that this model is a human. At least in the sense that it will understand and interpret natural language in the same way a human would. If I ask ChatGPT to write some prose, it is effectively the same as a ghostwriter. If I ask Dalle to generate an image, it's as if I have commissioned an artist to draw something for me. In reality, there are huge gaps between AI and human, and the anthropomorphism is harmful to effective HAI interaction.

The second part, that genAI is hard to steer using only prompt changes, reflects a need for change at the model and its encompassing interface. Prompting strategies like in-context learning are only a bandaid.


When is a Tool a Tool? User Perceptions of System Agency in Human–AI Co-Creative Drawing

I was a bit confused about the paper's delineation of "tool" vs. "agent". It seems like this was subjective and for the subject to decide. For example, one participant said “I would say tools are something that you use to make something that you have in mind." I don't think this is necessarily not an agent either, though. I think the delineation has more to do with initiative and capability. It also seems like "collaborator" and "agent" are sometimes used interchangably, which I think is imprecise.

Seeing as the overall goal of the paper was to use Reframer to examine what makes users perceive an AI as a tool vs. agent vs. collaborator, I guess the fluidity of definitions is intentional. However, the conclusion that there is some type of coupling between collaborative and agentic AI is surprising to me, since I think there are some very clear examples (coding agents) that are clearly agentic but not a collaborator in the sense that it will never take it up itself to code something that it feels like. It is only there to execute what you want, but it can take initiative to edit and run code as it sees fit.


GenTune: Toward Traceable Prompts to Improve Controllability of Image Refinement in Environment Design

Fine prompt tuning for image generation is a really hard task especially looking at the models used by the participants. These are state-of-the-art image gen models (Dalle, midjourney, stable diffusion) but they generate whole images. This does not align with a traditional designer workflow, which is typically done in layers so that elements can be edited in isolation. 

If I'm thinking of my dream image generation interface, I would want to understand which parts of my prompt correspond to which generated elements; this aligns with the survey results from the paper. I would be curious to see how image gen tools like those from Adobe or Canva or even Qwen Image Layered that would decrease frustrations from imprecise image editing.
